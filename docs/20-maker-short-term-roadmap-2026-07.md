## 主线：阶段 3 v0 实盘时间片 A/B（唯一晋级项）

这是本期唯一以"晋级/否决"为产出的工作，其余全部为并行支线。步骤：

1. **Gate 重验**：策略代码已合并，按 [14-maker-live-gate.md](14-maker-live-gate.md)
   完成一次受监督 canary 重验并留证。
2. **授权**：按 [19-maker-stage2-live-ab-runbook.md](19-maker-stage2-live-ab-runbook.md)
   记录精确授权文本，风险预算沿用 canary 口径（单 symbol、一档、最小量）。
3. **执行**：baseline（冻结静态配置）与 candidate（size skew 开启）交替 4h 臂，
   wind-down 换臂；比较窗口必须同时覆盖平静与趋势时段，趋势时段不足则延长采集而不是提前判定。
4. **判定**：按 18 号文档阶段 3 验收标准预注册判据出报告，核心是
   `p95 |position|` 降 ≥15%（或 ≥70% max_position 时间降 ≥25%）、net PnL ≥ 基线 95%、
   双边 uptime 降幅 ≤3pp、主动退出次数与 taker exit cost 不高于基线。

**判定后的分支**：

- **accepted** → candidate 配置成为新基线；下一个 alpha 候选从设计储备重新立项（不默认续做 v1）。
- **rejected（uptime/PnL 代价过大）** → 评估 v1"更陡但不停"的非线性 price skew 中间档，
  作为同阶段增量候选重走同一验收。
- **rejected（无效果）** → 阶段 3 回设计储备，短期内不再消耗 live 时间片；转向支线收尾。

## 低成本风险规则（来自 2026-07-20 做市调研，先手册后代码）

调研结论（详见当日调研记录）中两条与当前亏损画像直接相关、且实现成本极低：

- **宏观事件窗口**：CPI / FOMC / 非农公布前 15 分钟至后 30 分钟，手动停机或显著拉宽。
  依据：事件瞬间的盘口挂单是逆向选择重灾区，与我们"第一刀不在库存机制射程内"的
  仲裁结论互补——库存机制管不了的第一刀，可以用日历躲掉一部分。
- **周末缩量**：周末量降 40–60%、点差扩一倍，薄盘口下挂单更容易被单笔大单碾过。
  周末将 size 或运行时段收缩。

**执行方式**：本期只做"运维规则"——写入 runbook、人工执行、在遥测里标注事件窗口，
积累 4–8 周带标注的数据。自动化（CLI 侧日历配置 → typed input）只有在数据显示事件
窗口损耗确实可观时才立项，立项后按 18 号文档的 v0 流程走（默认关闭、replay 等价、A/B）。
在此之前不写代码，避免重蹈阶段 4"离线 credit 被 live 证伪"的覆辙。

## 设计储备（短期明确不做，附理由）

- **趋势滤波暂停报价**：与阶段 4 同族。live 证据已表明"漂移条件下加宽"为负
  （成交率损失 > 躲掉的毒性），"暂停"是更极端的形式，证据门槛只会更高。除非事件
  窗口标注数据给出新证据，不重启。
- **对冲腿（跨所对冲，Hyperliquid 平库存）**：教科书上治理库存风险的正解，也是
  skew 类机制的天花板替代。但涉及第二个 venue 的执行、资金和故障域，属于架构级
  变更，必须在安全轨二级完成之后才有资格立项。记录在此防止遗忘，不进短期计划。
- **SIP-5A 奖励层报价**（带内远挂、低风险吃奖励）：当前规模下奖励收入 ≈ 0，
  规模扩大前无经济意义。与扩规模决策绑定评估。
- **安全轨二级**：维持原计划——扩大规模（加 size / max_position / 多 symbol）前强制完成。
